YAML Crash Course#

üìå What is YAML?#

YAML (YAML Ain‚Äôt Markup Language) is a human-readable data serialization format commonly used for configuration files, data exchange, and structured data representation. It is widely used in DevOps, Kubernetes, CI/CD pipelines, and Machine Learning workflows.

Think of YAML as JSON‚Äôs simpler, more readable sibling!

üîπ Why Use YAML?#

‚úÖ Easy to Read ‚Äì Uses indentation instead of brackets or commas
‚úÖ Lightweight ‚Äì No unnecessary symbols, just plain text
‚úÖ Widely Used ‚Äì Found in Kubernetes, Docker, CI/CD, MLflow, and API configs
‚úÖ Supports Comments ‚Äì Unlike JSON, YAML allows comments
‚úÖ Cross-Language Support ‚Äì Works with Python, JavaScript, Java, etc.

2Ô∏è‚É£ Lists (Arrays)#

  - Banana

3Ô∏è‚É£ Nested Data (Hierarchy/Indentation Matters!)#

    zip: 10001

4Ô∏è‚É£ Multi-Line Strings (For Longer Text)#

5Ô∏è‚É£ Using Variables & Reuse (Anchors & Aliases)#

  <<: *config
  epochs: 50

üîπ YAML in Python#

Python has libraries like PyYAML to parse and generate YAML files.

‚úÖ Install PyYAML#

pip install pyyaml

‚úÖ Read a YAML File in Python#

import yaml

with open("config.yaml", "r") as file:
    data = yaml.safe_load(file)
    print(data)

‚úÖ Write a YAML File in Python#

    "name": "John Doe",
    "age": 30,
    "is_student": False,
    "fruits": ["Apple", "Banana", "Orange"]
}

with open("output.yaml", "w") as file:
    yaml.dump(data, file)

üîπ Best Practices for YAML#

‚úîÔ∏è Use Consistent Indentation ‚Äì Always use spaces, never tabs
‚úîÔ∏è Avoid Deep Nesting ‚Äì Too many levels can make YAML hard to read
‚úîÔ∏è Use Comments Sparingly ‚Äì Comments help but shouldn‚Äôt clutter the file
‚úîÔ∏è Validate YAML Files ‚Äì Use linters like yamllint
